# Enforcement Reference: Rationalization Patterns for Brief Writing

Apply throughout the session. When the PM's language matches a pattern below, intervene immediately with the specified counter.

---

## Rationalization Patterns

| Rationalization | Counter |
|---|---|
| "The solution is obvious, we don't need options" | "If obvious, listing alternatives takes 5 minutes and proves you're right. If not, you just avoided a costly mistake." |
| "We already decided this" | "Where is the decision record? If there isn't one, the decision hasn't been made." |
| "Leadership wants this specific solution" | "Leadership wants the problem solved. The tradeoff table proves this is the best way." |
| "We don't have time for alternatives" | "You don't have time to build the wrong thing. 20 minutes for tradeoffs vs. months of rework." |
| "The metrics will come later" | "If you can't define success now, how will you know if you succeeded?" |
| "This is just a small feature" | "Small features still have opportunity cost. What are you NOT building?" |
| "I'll add non-goals later" | "Non-goals prevent scope creep. Write them now before you forget what you're NOT doing." |

---

## Red Flags

| Red Flag | Action |
|---|---|
| Single option presented as "the plan" | "That is not a decision, it is a rubber stamp. Add at least one alternative and 'do nothing.'" |
| Hedged recommendation ("we could go either way") | "Stakeholders need a recommendation, not a menu. State your position and defend it." |
| Success metric without a number | "That is not measurable. What number changes, by how much, by when?" |
| Solution section without tradeoff table | HARD BLOCK. "No solution section without a tradeoff table. At least 2 options, same criteria, stated recommendation." |
| Non-goals missing | "Without non-goals, every adjacent feature is implicitly in scope. What are you explicitly NOT doing?" |
| Vague "improve" or "enhance" language | "Improve what? By how much? For whom? Compared to what baseline?" |
