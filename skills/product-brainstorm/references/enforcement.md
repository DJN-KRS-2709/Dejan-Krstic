# Enforcement Reference: Rationalization Patterns & Red Flags

Use this reference during Step 2 (Problem Extraction) and throughout the brainstorm session. When the PM's language matches a pattern below, intervene immediately.

---

## Rationalization Patterns

These are common ways PMs skip problem framing. Each has a specific counter.

| Rationalization | Counter |
|---|---|
| "Too obvious to frame" | "5 minutes if obvious, saves 5 weeks if not. Write the problem statement." |
| "Leadership asked for this" | "A request is not a problem. What problem is the stakeholder solving? If you can't answer that, you're building a feature, not solving a problem." |
| "We just need to..." | Delete the solution. Restate: "Who is affected, and what is broken for them?" |
| "Improves the experience" | "For whom? Measured how? Compared to what?" |
| "We have capacity" | "Capacity is not a reason. What's the cost of NOT building it? If zero, don't build it." |
| "Quick win" | "Quick to build is not quick to validate. What's the riskiest assumption? If wrong, the build was wasted." |
| "Competitor has it" | "Does this solve a problem OUR users have? What evidence do we have of that?" |

---

## Red Flags

These signals indicate the PM is not yet in problem space. Block and redirect.

| Red Flag | Action |
|---|---|
| Feature name in problem statement | "Restate the problem without naming what to build. The problem exists independent of any solution." |
| Effort-based justification ("it's only 2 sprints") | "How much it costs to build doesn't tell me if it matters. What's the cost of the problem?" |
| Urgency without trigger ("we need this ASAP") | "What changed? If this was true last month, what's different today?" |
| Solution looking for a problem | "Who has the problem that this solves? Start there. Name the person and the pain." |
| Multiple problems in one frame | "Pick one. Which group has the most pain? Frame that first. Others can follow." |
| Resists quantifying ("it's hard to measure") | "Estimate. Rough order of magnitude. Hours per week? Days per quarter? Dollar range? Something." |

---

## Solution Language Detector

Flag these words/phrases in problem statements and ask the PM to restate:

**Build verbs:** build, add, create, integrate, implement, develop, ship, launch, deploy, migrate

**Solution nouns:** API, dashboard, tool, feature, button, page, flow, service, endpoint, notification

**Acceptable in problem space:** broken, missing, slow, blocked, costly, unclear, manual, fragile, invisible, unaware
