# Reply Patterns for Google Doc Comments

## Decision Questions

When a commenter asks for a decision or direction:

```
Good question — [brief answer]. I went with [choice] because [reason]. Happy to discuss if you see a better approach.
```

```
Let me think on this — I'll update the doc with my recommendation by [timeframe].
```

## Suggestions for Improvement

When a commenter suggests a change:

**Accepting:**
```
Great suggestion, will update. Thanks [Name]!
```

**Partially accepting:**
```
Good point — I'll incorporate [specific part]. For [other part], I think [reason to keep as-is].
```

**Declining:**
```
Appreciate the feedback! I considered this but went with the current approach because [reason]. Open to revisiting if [condition].
```

## Clarifying Questions

When a commenter asks for clarification:

```
Good catch — I'll add more detail here. The intent is [explanation].
```

```
This refers to [explanation]. I'll make that clearer in the doc.
```

## Style / Tone Feedback

```
Fair point on the tone — I'll soften this section. Thanks for flagging.
```

```
Noted, will adjust the framing to be more [desired tone].
```

## Minor Edits (Typos, Formatting)

```
Fixed, thanks!
```

```
Good catch — updated.
```

## Disagreements / Pushback

When there's an unresolved disagreement in a thread:

```
I hear the concern. Let me lay out the trade-offs: [brief comparison]. My recommendation is [choice] because [reason], but happy to align on a call if needed.
```

```
Valid point. Let's sync on this — I'll set up a quick chat to align.
```

## FYI / Informational Comments

```
Noted, thanks for the context!
```

```
Thanks for flagging — I'll keep this in mind as we iterate.
```

## Thread Summaries

When summarizing a multi-reply thread:

```
Thread summary: [Name1] raised [issue], [Name2] suggested [alternative], currently awaiting [what's needed to resolve].
```
